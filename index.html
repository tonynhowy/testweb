Resultados

Use o seletor acima para filtrar os resultados por método (como `insert`, `remove, search`). Usando as setas na parte e baixo, se navega entre tempo de execução (em milissegundos por operação) e memória alocada pelo Garbage Collector do Java, em MB/sec entre BTree, SplayTree e TreeMap.

Análise

Discussão crítica dos resultados, com destaque para desempenho, consumo de memória e aplicações recomendadas para cada estrutura.

Conclusão

Recomendamos o uso da BTree para sistemas com alta carga e dados persistentes. Para cenários com acesso repetitivo, a SplayTree é uma escolha eficaz.
